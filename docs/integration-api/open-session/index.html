Open Session

In order to open a session, send a GET request to the open-session endpoint. The server will respond with an iframeURL that you can open in a new window, the current window, or embed in an iframe to display the BidPaw lobby.

The following diagram describes the interactions between a player, your casino, and BidPaw when a user opens the lobby for the first time after registration:

Endpoint URL​

GET https://API_URL/api/casino/open-session/?clientId={CLIENT_ID}&userId={USER_ID}&nick={optional_nickname}&lang={lang}&currency={currency}&country={country}&gameId={gameId}

signRequest signature, generated using your shared secret keyRequired

Request Parameters​

Parameter Behavior and Special Cases​

Language Parameter (lang)​

If an unsupported language code is provided in the lang parameter, no error will be returned and the interface in the iframe will automatically default to English (en).

Currency Parameter (currency)​

The currency parameter is required. If an unsupported currency is provided, the request will be rejected and an error response will be sent.
See the Error Handling section for error text and status code details.

Country Parameter (country)​

The country parameter is required.

Nickname Parameter (nick)​

  • If the nick parameter is not provided, a random nickname (ID) will be generated automatically for the user.
  • If a different nick is sent on a repeated request (for the same user), the user’s nickname will be updated.
    The system guarantees that nicknames are unique among all users:
    If the requested nickname is already in use, a random character (letter or digit) separated by _ will be appended to the end of the nickname, and this will be repeated until a unique nickname is created.

Allowed Countries (allowedCountries)​

  • Use the allowedCountries parameter to explicitly define which countries are permitted. Provide a comma-separated list of ISO country codes (e.g. allowedCountries=US,GB,BY,RU).
  • If the user's country is not in the allowed list, access will be denied and an error will be returned (the iframe will not open).

Parameter Validation and Error Handling​

This endpoint enforces strict validation of required parameters and supports descriptive error responses.
Below are all error scenarios, their status codes, and response formats.

1. Missing Required Parameters​

  • If any of the following parameters are missing or empty: clientId, userId, gameId, currency, country
  • Status code: 400
  • Response: 
    {
      "data": null,
      "status": 400,
      "message": "Required parameters missing: clientId, userId, gameId, currency, country"
    }
  • The message will contain a comma-separated list of missing parameter names.

2. Signature Validation​

  • If the sign header is missing or not a string:

  • Status code: 400

  • Response:

    {
      "data": null,
      "status": 400,
      "message": "Not valid request. Missing required sign header."
    }

  • If the sign header fails verification:

  • Status code: 400

  • Response:

    {
      "data": null,
      "status": 400,
      "message": "Request signature invalid"
    }

3. Unsupported Currency​

  • If the currency value is not among the supported currencies, the session will not be opened and an error is returned.
  • Status code: 400
  • Response: 
    {
      "data": null,
      "status": 400,
      "message": "User's currency {currency} is not supported."
    }
  • The {currency} placeholder will be replaced with the value that was sent.

4. Country Not Allowed​

  • If allowedCountries is set and the user's country is not in the allowed list, access will be denied and an error is returned.
  • Status code: 400
  • Response: 
    {
      "data": null,
      "status": 400,
      "message": "Service is currently not available in {country}."
    }
  • The {country} placeholder will be replaced with the value that was sent.

5. Internal Server Errors​

  • Any unexpected or unhandled error during request processing.
  • Status code: 500
  • Response: 
    {
      "data": null,
      "status": 500,
      "message": "{error message}"  
    }

Parameter Descriptions​

  • lang: Optional. Determines the UI language. If an unsupported language code is provided, the English interface (en) will be shown automatically..

  • currency: Required. Sets the player's currency for displaying bid pack prices. Must be one of the supported currency codes.

  • country: Required. Indicates the user's country. If the country is not allowed, access will be denied.

  • gameId: Required. A global identifier linking the provider, platform, and casino for session tracking and callbacks.

  • nick: Optional. If not provided, a random value will be generated. If changed in a repeated request, the user's nickname will be updated. If the requested nickname is already used by another user, the system will append a random character (with underscore) to ensure uniqueness.

  • allowedCountries: Optional. Comma-separated list of allowed country codes. If present, only users from these countries are permitted. If the user's country is not in this list, the session will not be opened and an error will be returned.
